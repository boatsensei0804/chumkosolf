# Sensitive Data Handling (PDPA)

ระบบนี้เก็บข้อมูลอ่อนไหวจำนวนมาก โดยเฉพาะ**ข้อมูลเด็ก**และ**เลขบัตรประชาชน 13 หลัก** ของครู นักเรียน และผู้ปกครอง ทุกครั้งที่โค้ดแตะข้อมูลเหล่านี้ต้องทำตามกฎด้านล่าง

## ข้อมูลที่ถือว่าอ่อนไหวในระบบนี้

- เลขบัตรประชาชน (national_id) — อ่อนไหวสูงสุด
- เลขบัตรประจำตัวราชการ (civil_servant_id)
- วันเดือนปีเกิด, ที่อยู่, เบอร์โทร, อีเมล
- รูปถ่าย และไฟล์แนบส่วนบุคคล (เอกสาร, เกียรติบัตร)
- ข้อมูลนักเรียนทุกอย่าง (เป็นข้อมูลเด็ก ระวังเป็นพิเศษ)

## กฎการเก็บ (Storage)

1. **เลขบัตรประชาชนต้องเข้ารหัสตอนเก็บ** (encryption at rest) หรืออย่างน้อยจำกัดสิทธิ์เข้าถึงระดับ column เข้มงวด เลือกแนวทางให้สม่ำเสมอทั้งโปรเจค
2. ห้ามเก็บข้อมูลอ่อนไหวใน log, error message, หรือ cache ที่ไม่ได้ป้องกัน
3. ไฟล์แนบเก็บใน storage ที่เข้าถึงผ่าน **signed URL ที่หมดอายุ** ไม่ใช่ public bucket

## กฎการส่งออก (API Response)

1. **ไม่ส่งเลขบัตรประชาชนเต็มกลับ frontend ถ้าไม่จำเป็น** — ส่งแบบ mask (เช่น `1-2345-xxxxx-xx-1`) สำหรับการแสดงผลทั่วไป
2. ส่งเลขเต็มเฉพาะ endpoint ที่จำเป็นจริง และเฉพาะผู้มีสิทธิ์ (เช่น กลุ่มบุคคลแก้ข้อมูลครู)
3. ไฟล์แนบส่งเป็น signed URL ไม่ใช่ path จริงใน storage

## กฎการเข้าถึง (Access Control)

จำกัดตามกลุ่มงาน + role เสมอ (ดูคู่กับ tenant-term-scoping):
- ข้อมูลครู/บุคลากร → กลุ่มงานบุคคล (+ admin)
- ข้อมูลนักเรียน/ผู้ปกครอง → กลุ่มงานวิชาการ (+ admin)
- ครูทั่วไปเห็นได้เฉพาะข้อมูลที่จำเป็นต่อหน้าที่ (เช่น ครูที่ปรึกษาเห็นนักเรียนในห้องตัวเอง) ไม่ใช่ข้อมูลส่วนบุคคลทั้งหมด

## Audit Log (บังคับ)

ทุกการ **ดู/แก้/ลบ** ข้อมูลส่วนบุคคลต้องบันทึก audit log:
```sql
audit_logs (
    id, school_id,
    actor_user_id,        -- ใครทำ
    action,               -- view, update, delete, export
    target_type,          -- personnel, student, guardian
    target_id,            -- ข้อมูลของใคร
    detail,               -- field ที่แตะ (ไม่เก็บค่าจริงของข้อมูลอ่อนไหว)
    created_at
)
```
- เก็บว่า "ใครเข้าถึงข้อมูลของใคร เมื่อไหร่" แต่ **ไม่เก็บค่าจริง** ของข้อมูลอ่อนไหวใน detail
- การ export/ดาวน์โหลดข้อมูลจำนวนมากต้อง log เป็นพิเศษ

## Frontend

- แสดงเลขบัตรประชาชนแบบ mask เป็น default ปุ่ม "แสดงเต็ม" เฉพาะผู้มีสิทธิ์ + log การกด
- ไม่ log ข้อมูลส่วนบุคคลลง console หรือ error tracking (Sentry ฯลฯ)
- ไฟล์แนบโหลดผ่าน signed URL ที่ backend ออกให้

## Checklist ก่อนเสร็จ

- [ ] เลขบัตรประชาชนเข้ารหัส/จำกัดสิทธิ์ตอนเก็บ
- [ ] response mask ข้อมูลอ่อนไหวที่ไม่จำเป็น
- [ ] เข้าถึงจำกัดตามกลุ่มงาน + role
- [ ] มี audit log สำหรับ view/update/delete
- [ ] ไฟล์แนบใช้ signed URL
- [ ] ไม่มีข้อมูลอ่อนไหวรั่วใน log/cache/error
- [ ] มี test ยืนยันว่าผู้ไม่มีสิทธิ์เข้าถึงข้อมูลส่วนบุคคลไม่ได้
